Autoencoders

Autoencoders are a type of ANN used to learn efficient codings of unlabeled data. Autoencoders learn two functions: and encoding function \(h=f(\mathbf{x})\) that transforms the input data, and a decoding function \(r=g(\mathbf{h})\) that recreates the input data from the encoded representation (i.e. it attempts to copy its input to its output). The goal is not for the autoencoder to learn exact mappings (i.e. \(g(f(\mathbf{x}))=\mathbf{x}\)), but to learn an approximate model by prioritising which aspects of the input should be copied and thus (hopefully) learning the most useful properties of the data.

Autoencoders can be trained in the same way as feed-forward neural networks, though may also be trained using recirculation.

Undercomplete Autoencoders

Constraining the hidden layer \(\bf{h}\) of an autoencoder to be of lower dimension than some input \(\bf{x}\) forces the autoencoder to learn the most important features of the input data. When the dimensions of the hidden layer are smaller than that of the input, the autoencoder is referred to as undercomplete.

The learning process is described simply as minimising a loss function \(L (\bf{x}, g(f(\mathbf{x})))\), where \(L\) is a loss function penalizing \(g(f(\bf{x}))\) for being dissimilar from \(\mathbf{x}\), such as the mean squared error.
