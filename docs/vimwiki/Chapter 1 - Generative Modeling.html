Chapter 1 - Generative Modeling

"Generative modeling is a branch of machine learning that involves training a model to produce new data that is similar to a given dataset."

Generative models must also be probabalistic rather than deterministic, since we want to be able to sample many different variations of the output, rather than get the same output every time.

While discriminative modeling estimates \(p(y|x)\), generative modeling estimates \(p(x)\) (or \(p(x|y)\) to model the conditional probability of seeing an observation \(x\) with a specific label \(y\)).

The Generative Modeling Framework

  • We have a dataset of observations \(\mathbf{X}\).
  • We assume that the observations have been generated according to some unknown distribution, \(P_{data}\).
  • We want to build a generative model \(P_{model}\) that mimics \(P_{data}\). If we achieve this goal, we can sample from \(P_{model}\) to generate observations that appear to have been drawn from \(P_{data}\).
  • There, the desirable properties of \(P_{model}\) are:
    • Accuracy - If \(P_{model}\) is high for a generated observation, it should look like it has been drawn from \(P_{data}\). If \(P_{model}\) is low for a generated observation, it should not look like it has been drawn from \(P_{data}\).
    • Generation - It should be possible to easily sample a new observation from \(P_{model}\).
    • Representation - It should be possible to understand how different high-level features in the data are represented by \(P_{model}\).

Generative Model Taxonomy

Different types of generative models take different approaches to modelling the density function \(p_\theta(\mathbf{x})\):

  1. Explicitly model \(p_\theta(\mathbf{x})\) but constrain the model so that the density function is tractable.
  2. Explicitly model a tractable approximation of \(p_\theta(\mathbf{x})\).
  3. Implicitly model the density function through a stochastic process that directly generates data.
